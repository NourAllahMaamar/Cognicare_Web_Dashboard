## Findings

1. **Invalid login returns HTTP 500**  
   Observed: `POST http://localhost:5173/api/v1/auth/login` (proxied to backend) → **500**. For wrong password/email, APIs typically return **401** with a stable JSON error. A 500 masks client error handling and confuses monitoring.

2. **Specialist creator routes without authentication**  
   `App.jsx` registers `/specialist/activities`, `/specialist/pecs/create`, `/specialist/teacch/create`, and `/specialist/skill-tracker` **outside** `SpecialistLayout`. Unauthenticated users can load these UIs (verified for activities, PECS, TEACCH). If saving calls API with tokens, saves may still fail server-side — but exposing the UI is a **policy leak** (data entry, UX confusion, possible client-side-only assets).

3. **Org login “Specialist Login” navigation**  
   During one click from org login, URL moved to `/specialist/login` while snapshot still showed org copy briefly — may be animation or overlapping layout. Worth a quick visual/regression check in a real browser.

4. **PECS phase labels in accessibility tree**  
   Combobox options show mixed quote characters (e.g. `\"“` in snapshot). Verify source strings in locales/components for **typographic quotes** vs encoding issues (may affect screen readers only).

5. **Default backend port documentation**  
   `vite.config.js` defaults proxy to `3000`; `config.js` mentions `3001` in a comment for another scenario. Developers should set **`VITE_BACKEND_ORIGIN`** explicitly to match the running API (documented in `architecture/API-AND-ENV.md`).

## Recommended fixes (prioritized)

### P0 — Security / product

- **Route guards** for `/specialist/activities`, `/specialist/teacch/create`, `/specialist/skill-tracker`, `/specialist/pecs/create`, and `/specialist/ai-recommendations/:childId`: require specialist (or appropriate) session, consistent with `SpecialistLayout` routes. Implementation options: wrap in a `ProtectedRoute` + role check, or nest under `/specialist/dashboard/...` with redirects.

### P1 — Backend / API contract

- **Login errors**: return **401** for invalid credentials, **422** for validation, reserve **500** for true server faults. Ensure the web app surfaces the error message without treating 500 as generic network failure only.

### P2 — UX / QA

- Clarify **Org vs Specialist** login navigation (loading state or full page transition) to avoid duplicate headings during transition.
- Audit **PECS** phase strings for quote consistency in `src/locales` and PECS creator.

### P3 — Engineering hygiene

- Remove or archive `*_OLD.jsx` files if unused, or document why they remain (see `STACK-AND-CONVENTIONS.md`).
- Add **E2E** later (Playwright): smoke login + one CRUD path per role when test accounts exist.
