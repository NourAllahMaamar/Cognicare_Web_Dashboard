# Bilan (session reports)

Each file here is a **bilan** — a dated record of QA runs, MCP-driven exploration, or other actions on the web app.

**Naming:** `YYYY-MM-DD-short-topic.md` (e.g. `2026-03-24-mcp-smoke.md`).

**Suggested sections:**

1. Environment (git commit if known, dev URL, backend URL, `VITE_BACKEND_ORIGIN`)
2. Scope (which roles/routes were tested)
3. Matrix or checklist (pass / fail / blocked)
4. Findings (bugs, UX, console/network)
5. Recommended fixes (prioritized)
6. Next development ideas

Stable architecture reference remains in the parent [`../README.md`](../README.md).
